üì¶ Installation

Clone from GitHub and install from source. Requires Python 3.10+.

# 1. Clone repository git clone https://github.com/bensargotest-sys/trustscore.git cd trustscore # 2. Create virtual environment (recommended) python3 -m venv venv source venv/bin/activate # Windows: venv\Scripts\activate # 3. Install package pip install -e . # 4. Seed database with 200+ MCP servers python scripts/seed_database.py # 5. Run MCP server python -m src.server # 6. Verify it works python -m tests.test_basic

MCP Client Configuration

Add to your MCP client (Claude Desktop, Cursor, etc.):

{ "mcpServers": { "trustscore": { "command": "python", "args": ["-m", "src.server"], "cwd": "/path/to/trustscore" } } }

FAQ

How do I install TrustScore?
Clone from GitHub and install from source:
git clone https://github.com/bensargotest-sys/trustscore.git
cd trustscore && pip install -e .
Then configure your MCP client (see Installation section above).
Is this a web API?
No. TrustScore is a local Python MCP server that runs on your machine. It's not a cloud service, web API, or SaaS product. All data stays local.
What data does TrustScore collect?
TrustScore runs locally on YOUR machine. Your data stays on your machine. Only if you choose to report interactions will that data be stored in your local SQLite database. There is no cloud service, no tracking, no data sent anywhere.
How do I use it with Claude Desktop?
Add TrustScore to your claude_desktop_config.json file (see Installation section). Restart Claude Desktop. You can then say "Check the trust score for github-integration-009" and Claude will use the MCP tool.
Is TrustScore free?
Yes! 100% free and open source (MIT license). No API keys, no rate limits, no costs. Download, use, modify, contribute.
How do trust scores work?
Trust scores (0.0-1.0) are calculated from actual success/failure rates, latency, and reliability over time. 0.9+ is excellent, 0.7-0.89 is good, 0.5-0.69 is acceptable. Scores also include risk flags like recent_failures, high_latency, etc.
